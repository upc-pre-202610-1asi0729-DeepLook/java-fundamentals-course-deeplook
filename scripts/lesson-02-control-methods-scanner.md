
# Lección 2: Condicionales, bucles, métodos y entrada de datos
## Descripción

En esta lección aprenderás a crear programas más dinámicos en Java usando estructuras de control como `if-else`, bucles `for` y `while`, métodos con parámetros y entrada de datos mediante `Scanner`.

## Objetivo de la lección

Al finalizar esta lección, el estudiante podrá:

- Usar `Scanner` para recibir información desde la consola.
- Aplicar `if-else` para tomar decisiones.
- Usar bucles `for` y `while` para repetir instrucciones.
- Crear métodos para organizar y reutilizar código.
- Usar parámetros y valores de retorno en métodos simples.

## Secuencia del video

### 1. Entrada de datos con Scanner

`Scanner` permite leer datos escritos por el usuario desde la consola. En esta parte se muestra cómo pedir información, guardarla en una variable y usarla dentro del programa.

Ejemplo:

- Pedir el nombre del usuario.
- Guardar el dato en una variable `String`.
- Mostrar un saludo personalizado.

Archivo relacionado:

- [ScannerNameExample.java](../completed-examples/lesson-02-control-methods-scanner/ScannerNameExample.java)

---

### 2. Condicional `if-else`

La estructura `if-else` permite que un programa tome decisiones según una condición. Esto ayuda a que el programa responda de forma diferente dependiendo de los datos ingresados.

Ejemplo:

- Pedir la edad del usuario.
- Verificar si es mayor o menor de edad.
- Mostrar un mensaje según el resultado.

Archivo relacionado:

- [IfElseAgeExample.java](../completed-examples/lesson-02-control-methods-scanner/IfElseAgeExample.java)

---

### 3. Bucle `for`

El bucle `for` sirve para repetir instrucciones cuando sabemos cuántas veces queremos que se repita una acción.

Ejemplo:

- Mostrar números del 1 al 5.
- Repetir un mensaje varias veces.
- Usar una variable contadora.

Archivo relacionado:

- [ForLoopExample.java](../completed-examples/lesson-02-control-methods-scanner/ForLoopExample.java)

---

### 4. Bucle `while`

El bucle `while` repite instrucciones mientras una condición sea verdadera. Es útil cuando no sabemos exactamente cuántas veces se repetirá una acción.

Ejemplo:

- Pedir una contraseña.
- Repetir la solicitud mientras la contraseña sea incorrecta.
- Permitir el acceso cuando la contraseña sea correcta.

Archivo relacionado:

- [WhilePasswordExample.java](../completed-examples/lesson-02-control-methods-scanner/WhilePasswordExample.java)

---

### 5. Métodos sin retorno

Un método sin retorno permite organizar el código y evitar repetir instrucciones. En Java, se usa `void` cuando el método realiza una acción, pero no devuelve un resultado.

Ejemplo:

- Crear un método `saludar`.
- Enviar un nombre como parámetro.
- Llamar al método varias veces con diferentes nombres.

Archivo relacionado:

- [VoidMethodExample.java](../completed-examples/lesson-02-control-methods-scanner/VoidMethodExample.java)

---

### 6. Métodos con retorno

Un método con retorno permite realizar una operación y devolver un resultado al programa. Esto ayuda a reutilizar lógica y mantener el código más ordenado.

Ejemplo:

- Crear un método `sumar`.
- Enviar dos números como parámetros.
- Retornar la suma de ambos números.

Archivo relacionado:

- [ReturnMethodExample.java](../completed-examples/lesson-02-control-methods-scanner/ReturnMethodExample.java)

---

## Conclusiones clave

- Las estructuras de control permiten que un programa tome decisiones y repita acciones.
- Los métodos ayudan a organizar y reutilizar código.
- Los parámetros permiten enviar datos a un método.
- Los métodos con retorno permiten obtener resultados a partir de una operación.
- `Scanner` permite recibir información escrita por el usuario desde la consola.
- Estos conceptos son la base para crear programas más interactivos en Java.
